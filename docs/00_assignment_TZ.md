## 2) Что именно нужно сдать (формат)
Финальный продукт:
- **Jupyter Notebook** с:
  - комментариями и промежуточными выводами в **Markdown**-ячейках,
  - кодом в code-ячейках;
- **файлы с данными**, которые использовались в проекте.

Требования к оформлению ноутбука:
- Использовать **заголовки и подзаголовки** в Markdown, чтобы было чётко видно выполнение требований по каждому этапу.
- В ноутбуке должны быть **промежуточные выводы по ходу исследования** в Markdown-ячейках.
- Проект **не должен** выглядеть как «голая тетрадка с кодом».
- **Распределение задач внутри команды — равномерное**.
- В начале ноутбука: краткий список «кто что сделал» по каждому участнику.

## 3) Требования к датасету(-ам)
- Размер выборки:
  - минимум **300 строк** (объектов) для команды из **3 человек**;
  - минимум **1000 строк** для команды из **4 человек**.
  - Чем больше — тем лучше.
  - В зависимости от темы объём может быть скорректирован преподавателем.
- Колонки (признаки):
  - минимум **5**,
  - лучше ориентироваться на **8–10+**,
  - желательно наличие признаков **разных типов** (количественные, категориальные и т.д.).
- Разрешено объединять несколько датасетов через `pandas.merge` / `pandas.concat`.

## 4) Этапы проекта и оценивание (всего до 10 баллов)

### Этап 0 — Парсинг данных (1 балл, необязательный)
Если парсинг **не выполняется** (берёте готовый датасет), **максимальная оценка за проект может быть 9**.

Сценарии и баллы:
- **1 балл**: проход по нескольким страницам сайта + по страницам каждого объекта (каждый объект → отдельная строка).
  - Парсинг включает **широкий** сбор различной информации.
  - Итоговая таблица: значительное число строк (≥ 300/1000) и столбцов (≥ 5 информативных).
- **0.5 балла**: проход по нескольким страницам сайта **без** посещения страницы каждого отдельного объекта.
- **0.25 балла**: сбор информации с небольшого числа страниц / использование готовых таблиц на сайтах.

### Этап 1 — Предварительная обработка (1.5 балла)
Требуется:
- Проанализировать столбцы: что содержится, какие типы данных, есть ли пропуски.
- Обработать:
  - **пропуски**,
  - **дубликаты**,
  - **выбросы**.
- Если в столбце пропусков большинство — его можно удалить.
- Для пропусков/выбросов: решить, что делать:
  - удалить или оставить,
  - заполнить (медианой/средним/иным),
  - обязательно **объяснить** выбор решений.

### Этап 2 — Создание новых признаков (1.5 балла)
Создать новые признаки на основе существующих (логично вписанные в ваш датасет).
- Пример: из даты получить «рабочий/выходной день», «конец квартала», «праздник» и т.д.
- Минимум: **2 новых признака** (больше — лучше).

### Этап 3 — Разведывательный анализ данных (EDA) (2.5 балла)
Применить методы с семинаров:
- `groupby`, фильтрации по условиям, `describe` и т.д.
- Построить **корреляционную матрицу**.
- Попробовать найти зависимости.
- Чем более интересные находки — тем выше балл.

### Этап 4 — Визуализация (2.5 балла)
Построить «красивые и интересные» визуализации, позволяющие выдвигать гипотезы.
- Минимум: **3 графика разных типов**.
- Можно использовать идеи из галереи `matplotlib`/`seaborn`, а также графики, обсуждавшиеся на семинаре.

### Дополнительный балл (до 1)
За:
- аккуратное оформление ноутбука,
- глубокий анализ,
- результаты, «превосходящие ожидания».
